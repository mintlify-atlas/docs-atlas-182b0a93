---
title: Backend client
description: Server-side Pipedream SDK client — create Connect tokens, run actions, deploy triggers, and manage connected accounts.
---

The backend client runs on your server and authenticates with the Pipedream API using your OAuth client credentials. It exposes all Connect API operations: token creation, account management, action execution, trigger deployment, and more.

<Warning>
  Never use the backend client in browser code. The `clientSecret` must remain server-side only.
</Warning>

## Create the client

```typescript
import { createBackendClient } from "@pipedream/sdk/server";

const client = createBackendClient({
  environment: "production",
  projectId: process.env.PIPEDREAM_PROJECT_ID!,
  credentials: {
    clientId: process.env.PIPEDREAM_CLIENT_ID!,
    clientSecret: process.env.PIPEDREAM_CLIENT_SECRET!,
  },
});
```

<ParamField body="environment" type="string" required>
  The project environment to use. Must be `"production"` or `"development"`. Accounts connected in `development` are isolated from `production`.
</ParamField>

<ParamField body="projectId" type="string" required>
  Your Pipedream project ID (e.g. `proj_abc123`). Find it in the [Pipedream dashboard](https://pipedream.com/projects).
</ParamField>

<ParamField body="credentials" type="object" required>
  OAuth credentials for your Pipedream API client.
  <Expandable title="credentials fields">
    <ParamField body="clientId" type="string" required>
      Your OAuth client ID.
    </ParamField>
    <ParamField body="clientSecret" type="string" required>
      Your OAuth client secret. Keep this server-side only.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="scope" type="string[]">
  Optional OAuth scopes to request when obtaining access tokens.
</ParamField>

## Connect tokens

### `createConnectToken`

Creates a short-lived Connect token for a user. Pass this token to your frontend so users can connect accounts and configure integrations.

```typescript
const { token, expires_at, connect_link_url } = await client.createConnectToken({
  external_user_id: "user-123",
  allowed_origins: ["https://yourapp.com"],
});
```

<ParamField body="external_user_id" type="string" required>
  Your user's ID in your own system. Any string that uniquely identifies the user.
</ParamField>

<ParamField body="allowed_origins" type="string[]">
  Browser origins allowed to use this token. Should match the origin of your frontend application.
</ParamField>

<ParamField body="success_redirect_uri" type="string">
  URL to redirect the user to after a successful account connection.
</ParamField>

<ParamField body="error_redirect_uri" type="string">
  URL to redirect the user to if the account connection fails.
</ParamField>

<ParamField body="webhook_uri" type="string">
  Webhook URL that Pipedream will call on connection success or failure.
</ParamField>

<ResponseField name="token" type="string">
  The short-lived Connect token. Pass this to your frontend SDK.
</ResponseField>

<ResponseField name="expires_at" type="string">
  ISO 8601 expiration time for the token.
</ResponseField>

<ResponseField name="connect_link_url" type="string">
  A hosted Connect Link URL you can share directly with users.
</ResponseField>

## Accounts

### `getAccounts`

Retrieves connected accounts for your project, optionally filtered by app or external user.

```typescript
const { data: accounts } = await client.getAccounts({
  external_user_id: "user-123",
  app: "slack",
});
```

<ParamField query="external_user_id" type="string">
  Filter by your user's ID.
</ParamField>

<ParamField query="app" type="string">
  Filter by app name slug (e.g. `"slack"`, `"github"`).
</ParamField>

<ParamField query="include_credentials" type="boolean">
  When `true`, includes the raw credentials (API keys, tokens) in the response. Use with care.
</ParamField>

<ParamField query="limit" type="number">
  Maximum number of records to return.
</ParamField>

<ParamField query="after" type="string">
  Cursor for pagination — retrieve records after this cursor.
</ParamField>

<ParamField query="before" type="string">
  Cursor for pagination — retrieve records before this cursor.
</ParamField>

### `getAccountById`

Retrieves a single connected account by its ID.

```typescript
const account = await client.getAccountById("apn_z8hD1b4");
```

```typescript
// With credentials
const account = await client.getAccountById("apn_z8hD1b4", {
  include_credentials: true,
});
console.log(account.credentials);
```

<ParamField path="accountId" type="string" required>
  The account ID (e.g. `apn_z8hD1b4`).
</ParamField>

<ParamField query="include_credentials" type="boolean">
  When `true`, includes raw credentials in the response.
</ParamField>

### `deleteAccount`

Deletes a connected account by ID.

```typescript
await client.deleteAccount("apn_z8hD1b4");
```

### `deleteAccountsByApp`

Deletes all accounts connected to a specific app for all users in the project.

```typescript
await client.deleteAccountsByApp("app_id");
```

### `deleteExternalUser`

Deletes all accounts and data for a specific external user. Use this when a user deletes their account in your system.

```typescript
await client.deleteExternalUser("user-123");
```

## Actions

### `runAction`

Runs a Pipedream action component on behalf of an external user.

```typescript
const result = await client.runAction({
  externalUserId: "user-123",
  actionId: "slack-send-message",
  configuredProps: {
    slack: {
      authProvisionId: "apn_z8hD1b4",
    },
    channel: "#general",
    text: "Hello from Pipedream Connect!",
  },
});

console.log(result.exports); // key-value pairs from $.export()
console.log(result.ret);     // return value
```

<ParamField body="externalUserId" type="string" required>
  Your user's ID. Must match the user who owns the connected account.
</ParamField>

<ParamField body="actionId" type="string" required>
  The component key (e.g. `"slack-send-message"`) or an object with a `key` field.
</ParamField>

<ParamField body="configuredProps" type="object" required>
  Props to pass to the action. The app prop must include `authProvisionId` set to the connected account's ID.
</ParamField>

<ParamField body="dynamicPropsId" type="string">
  ID of a previous dynamic props reload, if applicable.
</ParamField>

<ParamField body="stashId" type="string | boolean">
  ID of a File Stash to sync with the action's `/tmp` directory. Set to `true` to generate a new stash ID automatically. Files persist for up to 1 day.
</ParamField>

<ResponseField name="exports" type="unknown">
  Key-value pairs set by the action via `$.export(name, value)`.
</ResponseField>

<ResponseField name="os" type="unknown[]">
  Logs and observations emitted during execution.
</ResponseField>

<ResponseField name="ret" type="unknown">
  The return value of the action.
</ResponseField>

<ResponseField name="stashId" type="string">
  The File Stash ID used during execution, if one was provided or generated.
</ResponseField>

## Triggers

### `deployTrigger`

Deploys a Pipedream trigger component for an external user. The trigger listens for events and emits them to a webhook URL or workflow.

```typescript
const deployed = await client.deployTrigger({
  externalUserId: "user-123",
  triggerId: "github-new-issue",
  configuredProps: {
    github: {
      authProvisionId: "apn_z8hD1b4",
    },
    repoFullName: "my-org/my-repo",
  },
  webhookUrl: "https://yourapp.com/webhooks/github",
});

console.log("Deployed trigger ID:", deployed.data.id);
```

<ParamField body="externalUserId" type="string" required>
  Your user's ID.
</ParamField>

<ParamField body="triggerId" type="string" required>
  The trigger component key (e.g. `"github-new-issue"`) or an object with a `key` field.
</ParamField>

<ParamField body="configuredProps" type="object" required>
  Props to pass to the trigger, including the app's `authProvisionId`.
</ParamField>

<ParamField body="webhookUrl" type="string">
  URL to which the trigger will POST events.
</ParamField>

<ParamField body="workflowId" type="string">
  Pipedream workflow ID to receive trigger events.
</ParamField>

### `getTrigger`

Retrieves a deployed trigger by its ID.

```typescript
const trigger = await client.getTrigger({
  id: "dc_abc123",
  externalUserId: "user-123",
});
```

### `getTriggers`

Retrieves all deployed triggers for an external user.

```typescript
const { data: triggers } = await client.getTriggers({
  externalUserId: "user-123",
});
```

### `updateTrigger`

Updates a deployed trigger — change its active state, name, or configured props.

```typescript
await client.updateTrigger({
  id: "dc_abc123",
  externalUserId: "user-123",
  active: false,
});
```

### `deleteTrigger`

Deletes a deployed trigger.

```typescript
await client.deleteTrigger({
  id: "dc_abc123",
  externalUserId: "user-123",
});
```

### `getTriggerEvents`

Retrieves the most recent events emitted by a deployed trigger.

```typescript
const { data: events } = await client.getTriggerEvents({
  id: "dc_abc123",
  externalUserId: "user-123",
  limit: 10,
});
```

## Apps and components

### `getApps`

Retrieves the list of apps available in Pipedream, with filtering and sorting options.

```typescript
const { data: apps } = await client.getApps({
  q: "slack",
  hasActions: true,
  sortKey: "featured_weight",
  sortDirection: "desc",
});
```

### `getApp`

Retrieves metadata for a single app by its name slug or ID.

```typescript
const { data: app } = await client.getApp("slack");
```

### `getComponents`

Retrieves available components (actions or triggers) for an app.

```typescript
const { data: actions } = await client.getComponents({
  app: "slack",
  componentType: "action",
});
```

### `getComponent`

Retrieves metadata for a specific component by its key.

```typescript
const { data: component } = await client.getComponent({
  key: "slack-send-message",
});
```

## Proxy requests

### `makeProxyRequest`

Makes an authenticated request to a third-party API through Pipedream's proxy, using credentials stored for the given external user and account.

```typescript
const response = await client.makeProxyRequest(
  {
    searchParams: {
      external_user_id: "user-123",
      account_id: "apn_z8hD1b4",
    },
  },
  {
    url: "https://slack.com/api/conversations.list",
    options: {
      method: "GET",
    },
  },
);
```

## Workflow invocation

### `invokeWorkflow`

Invokes a Pipedream workflow via its HTTP interface.

```typescript
const response = await client.invokeWorkflow(
  "https://en-abc123.m.pipedream.net",
  {
    body: { message: "hello" },
  },
);
```

### `invokeWorkflowForExternalUser`

Invokes a workflow with the connected account credentials of a specific external user injected automatically.

```typescript
const response = await client.invokeWorkflowForExternalUser(
  "https://en-abc123.m.pipedream.net",
  "user-123",
  {
    body: { message: "hello" },
  },
);
```
