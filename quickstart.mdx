This guide walks you through creating a workflow that receives an HTTP request, transforms the data, and returns a response. No infrastructure setup required.

## Before you begin

You need a free Pipedream account. Sign up at [pipedream.com](https://pipedream.com) — no credit card required.

## Build your first workflow

<Steps>
  <Step title="Create a new workflow">
    Go to [pipedream.com](https://pipedream.com) and click **New Workflow**. Give it a name like `my-first-workflow`.
  </Step>

  <Step title="Add an HTTP trigger">
    Select **HTTP / Webhook** as your trigger. Pipedream generates a unique URL for your workflow. Any HTTP request sent to that URL will trigger the workflow.

    ```
    https://eo1234abcd.m.pipedream.net
    ```

    <Note>Your trigger URL is unique to your workflow. Copy it — you'll use it in a later step.</Note>
  </Step>

  <Step title="Add a code step">
    Click **+** to add a step, then select **Run custom code**. Choose **Node.js**.

    Replace the default code with the following to read data from the incoming request and export a transformed result:

    ```javascript
    export default defineComponent({
      async run({ steps, $ }) {
        const { name = "world" } = steps.trigger.event.body;

        // Export a value for downstream steps to use
        $.export("greeting", `Hello, ${name}!`);

        return {
          message: `Hello, ${name}!`,
          receivedAt: new Date().toISOString(),
        };
      },
    });
    ```

    Click **Test** to run the step. You'll see the exported values appear in the **Exports** panel.
  </Step>

  <Step title="Return an HTTP response">
    Add another code step and use `$.respond()` to send a response back to the caller:

    ```javascript
    export default defineComponent({
      async run({ steps, $ }) {
        $.respond({
          status: 200,
          headers: { "Content-Type": "application/json" },
          body: {
            greeting: steps.nodejs.greeting,
          },
        });
      },
    });
    ```

    <Tip>
      `steps.nodejs.greeting` references the `greeting` export from the previous step. Step names appear in the left panel — rename steps to keep references readable.
    </Tip>
  </Step>

  <Step title="Deploy and test">
    Click **Deploy**. Your workflow is now live. Send a test request:

    ```bash
    curl -X POST https://your-trigger-url.m.pipedream.net \
      -H "Content-Type: application/json" \
      -d '{"name": "Pipedream"}'
    ```

    You'll get back:

    ```json
    { "greeting": "Hello, Pipedream!" }
    ```

    Open the workflow's **Event History** tab to inspect every event with its full input, outputs, and logs.
  </Step>
</Steps>

## What you just built

- A **workflow** with an HTTP trigger and two code steps
- A step that reads from `steps.trigger.event` and exports data with `$.export()`
- A step that uses `$.respond()` to return a synchronous HTTP response
- Full observability: every event is logged and inspectable in the UI
